click_extra.tests.conftest.skip_windows_colors = MarkDecorator(mark=Mark(name='skipif', args=(False,), kwargs={'reason': 'Click overstrip colors on Windows'}))#

Skips color tests on Windows as click.testing.invoke overzealously strips colors.

See: https://github.com/pallets/click/issues/2111 and https://github.com/pallets/click/issues/2110

click_extra.tests.conftest.runner()[source]#
click_extra.tests.conftest.invoke(runner, monkeypatch)[source]#

Executes Click’s CLI, print output and return results.

If color=False both <stdout> and <stderr> are stripped out of ANSI codes.

Adds a special case in the form of color="forced" parameter, which allows colored output to be kept, while forcing the initialization of Context.color = True. This is not allowed in current implementation of click.testing.CliRunner.invoke(). See: https://github.com/pallets/click/issues/2110
